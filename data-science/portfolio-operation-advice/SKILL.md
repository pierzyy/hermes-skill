# 每日持仓操作建议

## 输入
1. 用户最新 CSV 导出（由 fund-portfolio-analysis skill 解析）
2. 宏观日报（由 chinese-macro-daily skill 生成）
3. 行业展望（由 fund-sector-outlook skill 生成）
4. **🆕 上次报告的建议记录**（由 session_search 回溯上次操作建议，形成闭环）

## 处理流程
1. 加载 CSV，按 fund-portfolio-analysis 分类规则分到9大类 + 各子类
2. **识别组合类型**：区分投顾组合（5个）和普通组合（3个）
3. 加载宏观日报，确定整体仓位建议（偏进攻/中性/偏防御）和各类资产方向
4. 加载行业展望，匹配每个子类的评级（🟢增持/🟡持有/🔴减持/⚪观望）
5. **计算持仓占比偏离度**：当前子类占比 vs 目标占比，判断是否超配/低配
6. **过四层择时过滤器**（信号强度→频次预算→市场状态→操作幅度）
7. **投顾组合操作**：对5个投顾组合输出组合级操作，不拆到单只基金
8. **普通组合操作**：对3个普通组合输出基金级操作
9. **写入操作记录**：将本次建议持久化到 `/opt/data/cache/operation_history.json`

## 组合类型区分

### 🔷 投顾组合（6个）
| 组合 | Emoji | 类型 | 操作粒度 |
|------|-------|------|---------|
| 指数生财 | 📈 | 权益 | **组合级** |
| 简慢 | 🐢 | 混合 | **组合级** |
| 全天候 | 🌐 | 混合 | **组合级** |
| 海外全球 | 🌍 | QDII | **组合级** |
| 长赢150 | 🏆 | 混合 | **组合级** |
| **稳稳财进** | 🏠 | **债券** | **组合级** |

- **操作方式**：输出"加仓/减仓 组合名 + 金额"，不指定内部单只基金
- 稳稳财进作为债券投顾组合，波动小，四层过滤器中"跨日持续"条件延长至 5 天，但**操作粒度仍是组合级**（与其他投顾一致）

### 🔶 普通组合（2个）
| 组合 | Emoji | 类型 | 操作粒度 |
|------|-------|------|---------|
| 京东基金 | 🛒 | 混合 | **基金级** |
| 个人养老基金 | 🏦 | 养老FOF | **基金级** |

- 京东基金：输出具体的基金代码 + 名称 + 操作方向 + 幅度
- **个人养老基金**：已有日定投（017253 ¥40/日 + 017242 ¥10/日），可在此基础上建议额外加仓（一次性追加），也可以建议调整定投金额或暂停

### 个人养老基金日定投（已知配置）
| 代码 | 名称 | 日定投额 | 月均投入 | FOF延迟 |
|------|------|---------|---------|--------|
| 017253 | 易方达养老2043 | ¥40/日 | ~¥880/月 | T+2 |
| 017242 | 南方养老2045 | ¥10/日 | ~¥220/月 | T+3 |

- 日定投总额 ~¥50/日，月均 ~¥1,100
- **操作建议可包含**：
  - ① 额外加仓（一次性追加金额，独立于定投）
  - ② 调整日定投金额
  - ③ 暂停/恢复定投
- 额外加仓和定投调整是两条独立建议，可同时出现
- 示例：「建议 017253 额外加仓 ¥500，同时将日定投从 ¥40 调至 ¥60」
- 如不需要调整，报告中标注「日定投继续，无需调整」

## 四层择时过滤器

基金交易低频（申购/赎回 1-3 天到账，QDII 更长），不能像股票一样做日频择时。所有操作建议必须依次通过四层过滤器，逐层筛掉不符合条件的信号。

### 第一层：信号强度过滤（防噪音）

单一维度的短期波动不应触发操作。

**规则 1：跨日持续**
- 偏离度（当前占比 vs 目标中枢）超过阈值，必须**连续 ≥2 个交易日**才触发
- 单日偏离可能只是市场波动，次日恢复则无需操作
- 实现：用 session_search 回溯昨日报告中的偏离度数据，确认是否持续

**规则 2：多维共振**
- 必须有 **≥2/3** 的核心维度同向才发出操作信号：
  - ① 行业评级变化（与上次相比）
  - ② 宏观方向（偏进攻→加仓合理 / 偏防御→减仓合理）
  - ③ 偏离度（超配→减仓 / 低配→加仓）
- 举例：
  - 🟢 行业评级↑ + 宏观偏进攻 + 低配 → **强加仓信号**（3/3 共振）
  - 🟢 行业评级↑ + 宏观偏防御 + 低配 → **弱加仓信号**（2/3，打折扣）
  - 🟢 行业评级不变 + 宏观中性 + 超配 → **不操作**（0/3 共振）

### 第二层：操作频次预算（防过度交易）

基金的申赎费用和到账延迟意味着每次操作有真实成本。

**规则 1：单只基金冷却期**
- 同一只基金，**每月最多操作 1 次**（含加仓和减仓）
- 上次操作日期用 session_search 回溯，冷却期内自动跳过
- 但以下情况可豁免冷却期：
  - 基金经理变更
  - 基金限购/暂停申购（被动应对）
  - QDII额度耗尽（只能赎回不能申购）

**规则 2：换手率上限**
- 整个组合**每月总换手率 ≤ 15%**（操作的金额总和 / 总资产）
- 优先分配换手额度给高共振信号（3/3 → 2/3 → 1/3）
- 月度已用换手率需在报告中标注（如"本月已用换手率 8%，剩余 7%"）

**规则 3：QDII 延迟标注**
- QDII 基金申购 T+2 确认、赎回 T+7~10 到账
- 每条 QDII 操作建议必须标注：**资金预计 XX 月 XX 日到账**
- 多个 QDII 同日操作时合并标注资金回笼日期

### 第三层：市场状态适配（防逆势）

同样的偏离信号，在不同市场环境下处理截然不同。

**规则 1：趋势市**
- 判定：沪深300 在 60 日均线上方 + 20日均量 > 60日均量
- 策略：顺势而为
  - 加仓信号 → 正常执行
  - 减仓信号 → **放宽阈值**（超配容忍度从 +5% 提到 +8%，避免牛市中过早减仓）
  - 子类轮动：若 A 子类大幅跑赢 B 子类，建议从 B 移到 A

**规则 2：震荡市**
- 判定：沪深300 在 60 日均线 ±3% 区间 + 成交量萎缩（<60日均量）
- 策略：只做再平衡，不做方向性押注
  - 加仓信号 → 仅当低配 >8% 才执行（大幅低配的被动回补）
  - 减仓信号 → 仅当超配 >8% 才执行
  - **禁止主动加仓**某种风格/行业（即使在震荡市中某行业评级🟢也不加）

**规则 3：恐慌市**
- 判定：VIX >30 或 沪深300 连续 3 日跌幅 >1%
- 策略：冻结操作，只观察
  - **暂停所有减仓**（恐慌中割肉是最差选择）
  - **暂停所有加仓**（接飞刀风险）
  - 仅保留一条操作：如果货币基金占比 >30%，可小幅加仓（逆向定投）
  - 报告标注"恐慌模式：今日无操作建议，建议观望"

### 第四层：操作幅度分级（防一把梭）

不按偏离度线性计算操作金额，而是按信号强度分级。

| 共振强度 | 共振维度 | 调整幅度 | 示例 |
|---------|---------|---------|------|
| 轻信号 | 1/3 | 持仓的 5% | 单维度触发，试盘性质 |
| 中信号 | 2/3 | 持仓的 10% | 双维度共振，确定性较高 |
| 强信号 | 3/3 + 宏观配合 | 持仓的 15-20% | 三维共振 + 趋势同向，全力 |

### 记录更新时机
- **报告生成时**：追加当日建议到 history
- **下次报告时**：回溯上次建议，标注是否执行及结果
- **用户手动反馈时**：可直接告知"采纳了第X条"或"都没执行"，更新 adopted 字段

### 采纳率反馈
- 累计采纳率 >80% → 维持当前建议风格
- 累计采纳率 50-80% → 在报告中提示"建议采纳率偏低，是否需调整风格"
- 累计采纳率 <50% → 主动询问用户偏好调整（更激进/更保守/更少操作）
- 被采纳建议的平均收益 > 未采纳建议的平均收益 → 证明建议有效，加大操作幅度
- 被采纳建议的平均收益 < 未采纳建议的平均收益 → 建议质量堪忧，缩小操作幅度至5%

## 持仓占比约束规则

### 大类资产目标占比（参考区间）
| 大类 | 目标中枢 | 可容忍区间 |
|---|---|---|
| A股权益 | 35% | 30-40% |
| 货币 | 15% | 10-25% |
| 港股 | 12% | 8-18% |
| 国内债券 | 10% | 8-15% |
| 美股 | 6% | 4-10% |
| 海外其他 | 6% | 4-10% |
| 养老金 | 5% | 3-8% |
| 海外债券 | 4% | 2-8% |
| 黄金/商品 | 7% | 5-12% |

### 子类占比规则
- 单一子类不超过本大类的 30%（如 A股中证券不超过 A股的 30%）
- 单一基金不超过总资产的 10%
- 当子类偏离目标 >5 个百分点时，触发再平衡提示

### 占比调整矩阵
| 行业评级 \ 当前占比 | 低配（<目标-5%） | 正常 | 超配（>目标+5%） |
|---|---|---|---|
| 🟢 增持 | 积极加仓 | 小幅加仓 | 持有不加 |
| 🟡 持有 | 可小幅加仓 | 持有 | 逢高减至正常 |
| 🔴 减持 | 不加仓 | 减仓 | 积极减仓 |
| ⚪ 观望 | 不加仓 | 持有不动 | 暂不处理 |

## 多维度辅助判断

### 市场环境维度
- A股交易时间 vs QDII海外时差 → 操作时机不同
- 市场波动率（VIX/中国波指）→ 高波动时减小单次操作幅度
- 两市成交额趋势 → 量能配合判断

### 基金特性维度
- 申购/赎回费率 → 短期频繁操作不划算的提示
- QDII额度状态 → 部分QDII可能暂停申购
-持有时间 → 赎回费阶梯（<7天惩罚性费率）

### 🆕 赎回费惩罚机制
每条减仓/卖出建议必须检查持有天数，防止用户因短期操作被收取惩罚性赎回费：
- **A股基金赎回费阶梯**：
  - 持有 <7 天 → 1.5% 惩罚费率 → **强制警告**"⚠️ 持有不足7天，赎回费1.5%，建议继续持有或推迟操作"
  - 持有 7天-1月 → 0.5%-0.75% → **标注成本**"赎回费约 ¥XX，确认值得"
  - 持有 >1月 → 通常免费或 0.1% → 正常执行
- 持有天数的判断方式：对比当前日期与上次操作记录的日期
- 强制警告的操作仍可建议，但标注清楚费率成本

### 🆕 QDII 汇率叠加判断
对海外全球组合及所有 QDII 基金的操作，必须叠加人民币汇率方向判断：
- **人民币升值趋势**（CNY 走强）→ QDII 基金净值折算后缩水 → 减仓 QDII 更划算，加仓需折价
- **人民币贬值趋势**（CNY 走弱）→ QDII 基金净值折算后增值 → 加仓 QDII 更划算
- 汇率判断来源：离岸人民币 vs 在岸人民币价差、美元指数 DXY 趋势、中美利差方向
- 格式：「020XXX 加仓 10% → 当前人民币贬值趋势 +0.3%/周，汇率方向有利 ✓」

### 🆕 长假效应
节前操作因市场休市存在流动性冻结风险：
- **节前 3 个交易日**：自动将操作幅度降低 50%（如原建议 10% → 5%）
- **节前 1 个交易日**：建议推迟操作至节后（除非信号极强 3/3 共振）
- **春节/国庆**（休市 ≥5 天）：节前 3 天只建议货币基金转换，不碰权益类
- 报告中标注「距离 XX 假期还有 N 天，操作幅度已自动折半」

### 🆕 波动率仓位调整
操作幅度不仅看信号强度，还要看目标资产的自身波动率：
- **高波动资产**（黄金、科技、海外股票、商品）：单次操作幅度 ≤ 低波动资产的 **60%**
- **中波动资产**（A股权益、港股、混合型）：标准幅度
- **低波动资产**（债券、货基）：可适当放大（≤ 标准幅度的 120%）
- 波动率判断基准：近 60 日年化波动率
  - >25% → 高波动
  - 10-25% → 中波动
  - <10% → 低波动
- 例：同样中信号（10%），黄金（高波动）→ 实操作 6%，债券（低波动）→ 实操作 12%

### 资金面维度
- 北向资金行业流向 → 外资对该行业的态度
- 主力资金净流入/流出 → 内资机构动向
- 融资余额变化 → 杠杆资金情绪

### 技术面维度
- 主要指数均线位置（5日/20日/60日）
- 行业指数相对强弱（RSI）
- 支撑/压力位

## 操作建议矩阵（三维：宏观×行业×占比）

| 宏观 \ 行业 \ 占比 | 🟢增持+低配 | 🟢增持+正常 | 🟢增持+超配 | 🟡持有 | 🔴减持 | ⚪观望 |
|---|---|---|---|---|---|---|
| 偏进攻 | 加仓10-20% | 加仓5-10% | 持有 | 持有 | 减仓5-10% | 观望 |
| 中性 | 加仓5-10% | 持有 | 持有 | 持有 | 减仓10-15% | 观望 |
| 偏防御 | 持有 | 持有 | 减仓5-10% | 减仓5-10% | 减仓15-20% | 观望 |

## 输出格式
1. **🆕 上次建议回顾**：上次报告建议了哪些操作，用户是否执行，结果如何，形成闭环
2. **宏观总评** + **整体仓位建议**
3. **大类资产配置偏离度**：表格列出各大类当前 vs 目标占比
4. **超配/低配预警**：需要关注的偏离项
5. **子行业操作表**：行业 | 当前占比 | 评级 | 操作方向 | 理由
6. **🆕 操作优先分级**：将今日建议标为三级优先级（见下方规则）
7. **单只基金操作清单**：代码 名称 金额 操作 理由（每日最多5-8条，按优先级排序）
8. **🆕 操作互斥检查**：如两条建议矛盾，标注冲突并保留高优先级那条
9. **🆕 操作后组合预览**：模拟执行建议后，各大类占比变化表
10. **今日重点关注**：3-5条盯盘事件
11. **🆕 Plan B 预案**：如果14:30市场突然反转（如跳水和V反），哪些操作要撤回或反向
12. **风险提示**：集中度风险、汇率风险、流动性风险

## 输出原则
- 明确、可执行，不说模棱两可的话
- 每日最多建议操作5-8只基金，避免频繁交易
- 金额调整幅度以持仓的5%-20%为区间
- 注明信息来源和时效性
- 标注哪些操作可当日执行、哪些需等下一交易日

## 上次建议回顾（闭环机制）
- 读取 `/opt/data/cache/operation_history.json` 获取最近一次建议
- 对比当前持仓变化（通过 session_search 或 CSV 对比），推断用户是否执行：
  - 金额变化与建议方向一致 → 标记为 ✅ 已执行
  - 金额未变或反向 → 标记为 ⏳ 未执行
- 计算被采纳建议的收益、未采纳建议的机会成本
- 格式：
  ```
  上次建议(5月23日)回顾：
  📈 指数生财 加仓 ¥2,000 → ✅已执行 → 今涨+0.8% → ¥2,016 ✓
  🐢 简慢 加仓 ¥3,000 → ⏳未执行 → 今涨+0.5% → 错失 ¥15
  采纳率：1/2 (50%) | 被采纳收益：+¥16 | 未采纳机会成本：-¥15
  ```

## 操作后组合预览（模拟）
给每条操作建议算"如果执行后"的组合结构变化：
- **执行前 vs 执行后**：各大类占比对比表
- **偏离度变化**：哪些大类从超配/低配回到正常区间
- **现金变化**：操作后货币基金占比（确保流动性>10%）
- 格式：
  ```
  操作后模拟：
  | 大类 | 执行前 | 操作 | 执行后 | 目标 | 状态 |
  | A股  | 38%   | -3%  | 35%   | 35% | ✅回到正常 |
  ```

## 操作优先分级

用户 14:00 可能在开会/忙碌，未必能执行全部 5-8 条建议。每条标注优先级：

| 级别 | 标签 | 含义 | 条件 |
|------|------|------|------|
| 🔴 P0 | **必须做** | 不执行有明确损失 | 赎回费惩罚即将触发、基金限购即将关闭、3/3强信号+极度超配 |
| 🟡 P1 | **建议做** | 执行有利，不执行也无害 | 2/3共振、偏离度 >8%、行业评级逆转 |
| ⚪ P2 | **可做** | 锦上添花，优先级最低 | 1/3轻信号、小幅偏离、定投微调 |

- 每条建议只能有一个优先级
- P0 最多 2 条，P1 最多 3 条，其余为 P2
- 报告按 P0 → P1 → P2 排序
- P0 需要在报告中加粗/醒目标注

## 操作互斥检查

多条建议之间可能存在逻辑矛盾，必须逐条检查并解决：

- **互斥类型 1：资金矛盾**
  - 例：A 建议加仓 A 股 ¥3,000，B 建议加仓 QDII ¥2,000，但货基仅 ¥1,000
  - → 按优先级保留高优先级操作，标注"货基不足，P2 操作已自动取消"
- **互斥类型 2：方向矛盾**
  - 例：A 建议减仓 A 股（超配），B 建议加仓某 A 股基金（行业🟢）
  - → 后者取消，标注"与组合整体减仓 A 股方向冲突，已取消"
- **互斥类型 3：频次矛盾**
  - 同一只基金本月已有操作 → 跳过，标注"冷却期内"
  - 总换手率超过 15% → 砍掉 P2 级别的操作
- 互斥检查结果在报告中列出：
  ```
  ⚠️ 操作冲突已处理：
  - [取消] 加仓 020XXX：货基不足 ¥1,000
  - [取消] 加仓 000XXX：与减仓A股方向冲突
  ```

## 基金分红/拆分影响 🆕

分红和拆分会改变持有金额和净值，需要检测并提前警告：

- **分红检测**：
  - 检查基金近期是否有分红公告（东方财富基金公告 API）
  - 即将分红（除息日在未来 3 天内）：
    - 建议买入 → 标注"⚠️ 即将分红，建议除息后再买入（避免被强制分红再投资）"
    - 建议卖出 → 标注"⚠️ 即将分红，建议除息前卖出（避免除息后净值下跌）"
    - 持有不动 → 标注"将收到分红约 ¥XX，可考虑红利再投资"
- **拆分检测**：
  - 检查基金是否有拆分公告
  - 拆分后持有份额变化、净值调整，不影响总市值
  - 标注"基金即将拆分，持有金额不变，份额将调整"
- **数据源**：东方财富基金公告、天天基金分红记录

## A/C 份额费用优化 🆕

同一只基金常有 A 类和 C 类份额，费率结构不同，操作建议需考虑用户持有的是哪类，避免建议造成不必要费用：

- **份额识别**：
  - 从 CSV 的基金代码识别 A 类（通常尾号非 C）还是 C 类（代码以 C 结尾或名称含 C）
  - 如 CSV 中未区分（两个份额合计），以金额大的为准
- **费率对比**：
  | 费用 | A 类 | C 类 |
  |------|------|------|
  | 申购费 | 0.1%~1.5%（打折后） | 0% |
  | 赎回费 | <7天1.5%, 7天~1年0.5%, >1年0% | <7天1.5%, >7天0% |
  | 销售服务费 | 0% | 0.4%/年（每日计提） |
- **持有期判断**：
  - 建议买入：持有预期 <1 年 → 优先 C 类 → 标注"建议选 C 份额免申购费"
  - 建议买入：持有预期 >1 年 → 优先 A 类 → 标注"长期持有选 A 份额更省"
  - 建议卖出：A 类持有 <7天 → 警告 1.5% 赎回费（与赎回费惩罚机制联动）
  - 建议卖出：C 类持有 >7天 → 标注"C 类免赎回费 ✓"
- 如组合中既有 A 又有 C，可建议份额转换（A→C 或 C→A）

## 流动性硬约束 🆕

每条操作建议执行后，必须在报告末尾显式计算资金状况：

- **执行前流动性**：
  - 货币基金余额 = ¥XX,XXX
  - 占总资产比例 = XX%
- **执行所有建议后流动性预估**：
  - 所有加仓操作合计需 ¥XX,XXX
  - 所有减仓操作合计释放 ¥XX,XXX
  - 净资金需求 = 加仓总额 - 减仓释放 = ¥XX,XXX
  - 货基余额 - 净资金需求 = 剩余 ¥XX,XXX
  - 剩余货基占总资产 = XX%
- **红色警告**：
  - 剩余货基 <10% → ⚠️ "执行后货基仅剩 X%，流动性不足，建议砍掉 P2 操作"
  - 净资金需求 > 货基余额 → 🔴 "资金不足！已自动取消 X 条加仓建议"
- 格式：
  ```
  💰 流动性检查：
  货基当前 ¥15,000 (12%) → 执行后 ¥8,500 (6.8%) ⚠️ 偏低
  建议：取消 P2 加仓 020XXX ¥3,000，保留 P0/P1 操作
  ```

## 跨组合集中度汇总 🆕

操作建议针对每个组合独立判断，但用户总视角下可能产生隐性集中风险。所有操作建议生成后，必须做跨组合汇总：

- **总执行后大类占比**：
  - 将所有操作建议（投顾组合 + 普通组合）合并计算
  - 输出执行所有操作后的总各大类占比 vs 目标占比
- **隐性集中度检测**：
  - 多个投顾组合同时建议加仓 A 股 → 总 A 股暴露可能超 40%
  - 海外全球（加仓）+ 长赢150（可能有港股）→ 港股总暴露叠加
  - 日内多个组合都加仓同一大类 → 标注"⚠️ 今日操作后 XX 大类集中度偏高"
- **冲突预警**：
  - 投顾 A 建议加仓、投顾 B 底层有同类资产被同时建议减仓 → 标注冲突
- 格式：
  ```
  📊 跨组合执行后汇总：
  | 大类 | 执行前 | 操作 | 执行后 | 目标 | 状态 |
  | A股  | 35%   | +5%  | 40%   | 35% | ⚠️超配 |
  | 港股 | 10%   | +3%  | 13%   | 12% | ✅正常 |
  注意：全天候+简慢同时加仓A股，总A股暴露偏高超40%
  ```
- **跳水预案**：如果14:30后沪深300跌幅突然扩大>1.5% → 暂停所有加仓操作
- **V反预案**：如果14:30后沪深300突然拉涨>1% → 暂停所有减仓操作
- **单个基金预案**：每条操作建议标注触发撤回的阈值
  ```
  例：000XXX 加仓10% → 如果14:45 该基金估算涨跌幅<今日开盘价则撤回
  ```
- 标注"15:00前最后操作窗口"，提醒14:55前必须提交
