### 触发流程（CSV 等待机制）
1. **14:00 触发** → 先检测交易日：非交易日或半日市（12:00已收市）→ 静默跳过（`[SILENT]`）。节前最后交易日（全日市）正常触发但标注长假效应
2. **发送提醒**：「今日基金日报时间，请发送 FundMonitor CSV 导出文件」
2. **等待 CSV**：监听用户消息，收到 CSV 文件 → 保存到 `/opt/data/cache/` → 立即进入执行流程
3. **第一次提醒（14:05）**：若 5 分钟内未收到 → 发送「还未收到 CSV，请发送后我立即开始（1/3）」
4. **第二次提醒（14:10）**：若仍未收到 → 发送「再提醒一次，发来 CSV 即可开始（2/3）」
5. **第三次提醒+取消（14:15）**：若仍未收到 → 发送「今日未收到 CSV，日报任务取消」→ 退出
6. **收到 CSV 后**：回复「收到，开始生成今日报告…」→ 进入 Step 1
- 任何时候收到 CSV 都立即中断等待，直接开始
- 周末/节假日 cron 仍触发但 Step 0 交易日检测会静默跳过

### Step 1: 接收并校验 CSV
- 用户通过微信发送 FundMonitor CSV 导出文件
- 保存到 `/opt/data/cache/daily_fund_report.csv`
- 校验 CSV 格式（必须有 组合/代码/名称/持有金额 列）
- 校验通过 → 回复「收到，开始生成今日报告…」
- **CSV 新鲜度检查**：
  - 导出日期 = 今日 → 继续
  - 导出日期 = 昨日（交易日）→ 继续，报告中标注「数据截止昨日收盘」
  - 导出日期 >1 个交易日 → 告知用户数据过期但仍继续（标注风险）

### Step 2: 基金状态巡检
- **🆕 优先使用预采缓存（v2.1）**：
  1. 先检查 `/tmp/fund_cache_status.json` 是否存在且距今 <30 分钟
  2. 若存在且新鲜 → 直接 `read_file` 读取，从中提取 fundBuyStatus、limit 等字段；跳过 curl 爬取
  3. 若不存在或过期 → 走原流程：加载 `fund-status-check` skill
- 检查全部基金的申购/赎回状态、经理变更、规模异动
- 输出异常基金列表（供后续操作建议参考，剔除不可操作的基金）
- **🆕 投顾组合状态**：对5个投顾组合额外检查组合整体申购状态（是否限购/暂停）

### Step 3: 持仓分析（文字输出，不出图片）
- 加载 `fund-portfolio-analysis` skill
- 按9大类+子类解析持仓
- 计算各大类/子类占比
- **仅输出文字分析**，不生成图片报告

### Step 4: 实时行情采集（14:00 盘中快照）
- 国内基金：批量拉取 fundgz 实时估值（天天基金 API）
- QDII 基金：批量拉取对应 ETF 实时价格（通过 qdii_estimator.py）
- 货币基金：取最新万份收益
- 计算每只基金的：
  - 当日盘中涨跌幅（fundgz.gszzl 或 ETF chg_pct）
  - 盘中市值 = 持有金额 × (1 + 涨跌幅/100)
  - 盘中盈亏 = 市值 - 持有金额
- 汇总各大类/子类的盘中涨跌
- **🆕 数据流桥接**：将实时行情写入结构化临时文件 `/tmp/fund_report_step4.json`，供 Step 6 行业研判直接读取具体涨跌幅（不再依赖自然语言传递）
- **🆕 API 速率限制**：fundgz 并发 ≤15 只/批（ThreadPoolExecutor workers=15），单只超时 8s，失败自动 Continue
- **⚠️ 文件命名约定**：各步骤写入不同临时文件以避免相互覆盖：Step 3+4 输出到 `/tmp/fund_report_step4.json`，Step 5 不要写入同名文件。Step 6 单独输出到 `/tmp/industry_outlook.json`。最终 JSON 汇总到 `/tmp/fund_report_live.json`

### Step 4.5: 极端行情熔断检测 🆕
在 14:00 实时数据采集完成后，立即判断是否需要触发熔断：

- **触发条件（任一满足即熔断）**：
  - 沪深300 当日跌幅 >5%
  - 创业板指 当日跌幅 >7%
  - VIX >40
  - 两市跌停家数 >200
  - A股触发实际熔断机制
  - **🆕 连续衰减**：连续 3 个交易日沪深300累计跌幅 >4%（钝刀割肉，情绪累积比单日暴跌更危险）
- **熔断模式下的行为**：
  - 跳过 Steps 5-7（宏观日报/行业研判/操作建议），直接生成简化版报告
  - 报告仅输出：
    ```
    一、极端行情警告
    [触发原因 | 关键数据]
    二、持仓冲击快照
    [各大类盘中跌幅 | 估算亏损金额]
    三、建议
    今日不建议任何操作，观望为主。
    如已持有黄金/债券/货基，暂不调整。恐慌中割肉是最差选择。
    ```
  - 报告中不要出现操作建议、行业评级等正常内容
- **恢复条件**：下一交易日收盘后不存在触发条件 → 恢复正常报告流程

### Step 5: 宏观日报
- **🆕 优先使用预采缓存（v2.1）**：
  1. 检查 `/tmp/fund_cache_market.json` 是否新鲜（<30min）→ 直接读取 Sina 指数、汇率、商品数据
  2. 检查 `/tmp/fund_cache_north.json` 是否新鲜 → 直接读取北向资金数据
  3. 缓存命中可跳过大量 browser_ 调用（~120s → ~2s）
  4. 缓存未命中或过期 → 照常走 `chinese-macro-daily` skill
- 加载 `chinese-macro-daily` skill（用于缓存未覆盖的部分：PMI/CPI/政策解读等）
- 收集货币政策、PMI、CPI、汇率、北向资金、南向资金、美债、VIX等

- **🆕 重磅事件日检测**：若当日为 FOMC 议息日 / 中国政治局会议 / 美国 CPI 发布日 / 非农发布日 → 触发「重磅事件日模式」：冻结行业研判和操作建议，报告标注「重磅事件日，建议等待结果，暂无操作建议」

### 🆕 Step 8 JSON 格式约束（渲染脚本兼容性 — 2026-05-28 验证）

渲染脚本 `/opt/data/scripts/render_fund_report.py` 对 JSON 字段类型有严格要求。以下格式错误会导致渲染崩溃：

| 字段 | 期望类型 | 错误示例 | 正确示例 |
|------|---------|---------|---------|
| `macro` | **list** of `{key, val, is_up, is_down}` | `{"indices":{...},"rates":{...}}` | `[{"key":"CSI300","val":"4889(-0.4%)","is_up":false,"is_down":true}]` |
| `alerts` | **list** of `{title, detail, level}` | `["纯文本警告"]` | `[{"title":"⚠️ QDII限购","detail":"","level":"⚠️"}]` |
| `portfolio_status.*.type` | **string**: `"buy"`/`"sell"`/`"hold"` | 缺少此字段 | `"type":"buy"` |
| `planb` | **string** (用 ` \| ` 分隔) | dict | `"CSI300跌>1.5%→暂停 \| 恒生科技跌>1%→撤回"` |
| `industries.*.tag` | **string** 含文本 | `"🟢"` | `"🟢增持"` |

**⚠️ 特别注意**：Step 5 delegate_task 输出的 JSON 中，`macro` 字段通常是一个包含 `indices`/`rates`/`fx` 等子对象的 dict，**必须先转换为 list 格式再传给渲染脚本**，否则渲染脚本第68行 `m.get('key', ...)` 会因 m 是字符串而崩溃。

转换方法（Python）：
```python
# 将 Step 5 的 macro dict 转为渲染器期望的 list 格式
macro_items = []
for sym, info in step5_data['indices'].items():
    macro_items.append({
        'key': info['name'], 
        'val': f"{info['current']:.0f} ({info['chg_pct']:+.1f}%)",
        'is_up': info['chg_pct'] > 0,
        'is_down': info['chg_pct'] < 0,
    })
# 同样转换 rates, fx, commodities, futures, pe 等
data['macro'] = macro_items  # ✅ 渲染器可正常迭代
```

**⚠️ 特别注意**：Step 5 delegate_task 输出的 JSON 中，`macro` 字段通常是一个包含 `indices`/`rates`/`fx` 等子对象的 dict，**必须先转换为 list 格式再传给渲染脚本**。否则渲染脚本第68行 `m.get('key', ...)` 会因 m 是字符串而报 `AttributeError: 'str' object has no attribute 'get'`。
